shell：针对 shell 设置 Ruby 版本，该设置将覆盖 global 和 local 设置。

性能测试心得体会篇(一)

发表于 2015-12-01   |  

前段时间接触了几个性能测试的项目，主要用到了loadrunner，这里把关于性能测试，Loadrunner的一些体会记录下来。

测试需求

在进行测试之前我们一定要确定测试目标，测试目标的确定一般来讲有以下方式

1.客户提供
2.根据网站访问数据进行估算（估算的方法可以使用二八模型-80%的操作是在20%的时间里完成的）
3.客户也不清楚，也无法提供网站流量数据，只能根据用户量进行大概估算了（就是测测看）

Loadrunner中的一些名词含义

事务：我们可以自己定义事务的范围，可以是一次首页访问，一次用户登录，一次用户注册。用户完成一个事务往往包含了多次http请求。

并发/VUSER:说白了就是开了多少个线程，loadrunner实现并行计算的策略是每50个线程开启一个进程，这里并发数和对服务器的压力是成正相关的，但并不代表并发数越高对服务器的压力就越大。
程序运行的时候根据不同的压力机配置设置恰当的线程数量，这时程序运行的效率最高，造成的压力在最高点。

注：平时跟项目经理和客户沟通的时候经常会提到多少多少并发，这里提到的并发和vuser的数量是不一样的，没有相关经验的人，提到并发直观的理解是有多少个用户在同时使用网站。这个数据可以根据脚本运行时间和通过的事务数算出来。

平均响应时间：一段时间内事务的平均响应时间。
每秒事务数：每秒完成的事务数。
吞吐率：每秒交互的流量总计。

项目注意事项

1.了解网站结构：要明确工程的部署情况。
例如：应用服务器，资源服务器，负载服务器的分部及协同工作的方式。

2.明确测试目标：大多数情况下客户或项目人员对性能方面不是特别了解，项目也没有正式投入使用，无法提供访问量等数据。这种情况就可以压一压看看网站情况.当然也要对不合理的测试要求进行沟通。
例如：要求对每个业务都进行负载测试，其实是完全没有必要的，而且费时费力的事情。

3.带宽要充足：能要多少要多少。

4.想到了后面再补

VPS简介

通常供应商会提供一个web管理界面，在这里通常可以完成系统重装，备份/还原，升级配置，续费…一些基本操作。

ShadowSocks配置

服务端：

过程

apt-get install python-pip
pip install shadowsocks
find / -name shadowsocks #看看装在哪里了
mkdir shadowsocks        #在etc目录下新建名为shadowsocks的文件夹
vim config.json          #新建名为config.json的文件写入一下内容

{
"server":"my_server_ip", #VPS的IP
"server_port":8388,         #VPS提供该服务的端口
"local_port":1080,         #shadowsocks客户端设置的端口，上网时需要把浏览器代理到这个端口

}

apt-get install python-m2crypto #为了支持以上加密方式
vim /etc/rc.local               #设置为开机自启动添加以下信息
/usr/local/bin/ssserver –c /etc/shadowsocks/config.json

在VPS的web管理界面重启VPS。

客户端：

1.下载客户端填写config.json文件中对应的信息。
2.使用 SwitchySharp（chrome代理管理插件）或FoxyProxy（Firefox代理管理插件）代理设置到对应local_port端口

DNS域传送漏洞学习记录

发表于 2015-10-12   |  

最近研究了一下DNS域传送漏洞,这里记录一下学习结果

DNS域传送漏洞概述

DNS服务提供了域名到IP的映射,服务端口通常为53.DNS服务器分为三类:主服务器,备份服务器,缓存服务器.其中备份服务器同步主服务器的过程为域传送.
当DNS服务器配置不当时可能导致任何匿名用户都可以获得整个域的所有记录,整个网络结构就暴漏给了攻击者.危害也就不言而喻了.

常见的DNS记录有以下几类

类别 说明
A ip地址映射
AAAA ipv6地址映射
CHAME 别名
NS 域名记录
MX 邮箱交换记录
PTR 反向记录
TXT 邮箱信息

nslookup命令检测域传送漏洞

1.非交互式

cmd> nslookup -type=ns baidu.com
#查看baidu.com此域名被哪个dns服务器解析
cmd> nslookup -type=ptr 8.8.8.8
#反向查看8.8.8.8的域名

2.交互式

cmd> nslookup #进入交互
cmd> server dns.baidu.com #指定dnsserver
cmd> ls baidu.com #列出所有记录

可以写一个bat脚本保存为 dnstest.bat

echo ls %1 | nslookup - %2

在使用时格式如下:

dnstest baidu.com dns.baidu.com
#param1中写入访问域名
#param2中写入dns地址

lock = threading.Lock()
c_index = 0
def test_DNS_Servers():
    global c_index
    while True:
        lock.acquire()
        if c_index >= len(colleges):
            lock.release()
            break    # End of list
        domain = colleges[c_index].lstrip('www.')
        c_index += 1
        lock.release()
        cmd_res = os.popen('nslookup -type=ns ' + domain).read()    # fetch DNS Server List
        print domain
        dns_servers = re.findall('nameserver = ([\w\.]+)', cmd_res)
        for server in dns_servers:
            if len(server) < 5: server += domain
            cmd_res = os.popen(os.getcwd() + '\\BIND9\\dig @%s axfr %s' % (server, domain)).read()
            if cmd_res.find('Transfer failed.') < 0 and \
               cmd_res.find('connection timed out') < 0 and \
               cmd_res.find('XFR size') > 0 :
                lock.acquire()
                print '*' * 10 +  ' Vulnerable dns server found:', server, '*' * 10
                lock.release()
                with open('vulnerable_hosts.txt', 'a') as f:
                    f.write('%s    %s\n' % (server.ljust(30), domain))
                with open('dns\\' + server + '.txt', 'w') as f:
                    f.write(cmd_res)           
threads = []
for i in range(10):
    t = threading.Thread(target=test_DNS_Servers)
    t.start()
    threads.append(t)
for t in threads:
    t.join()
